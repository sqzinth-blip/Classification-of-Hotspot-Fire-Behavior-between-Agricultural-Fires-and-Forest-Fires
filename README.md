# 🚀 การจำแนกพฤติกรรมจุดความร้อนด้วยเทคนิค ST-DBSCAN
### กรณีศึกษา: พื้นที่อำเภอบางระกำ และอำเภอเมืองพิษณุโลก จังหวัดพิษณุโลก

การวิเคราะห์ข้อมูลเชิงพื้นที่และเวลาขั้นสูง (Spatiotemporal Data Science) เพื่อวิเคราะห์ จัดกลุ่ม และจำแนกพฤติกรรมการเกิดจุดความร้อน (Hotspot) เปรียบเทียบระหว่างไฟเกษตรกรรมและไฟป่า โดยการบูรณาการข้อมูลการสำรวจระยะไกลจากดาวเทียม (Satellite Remote Sensing) ร่วมกับแผนที่การใช้ประโยชน์ที่ดิน เพื่อสร้างองค์ความรู้และการแสดงผลในรูปแบบเว็บแผนที่อินเทอร์แอคทีฟ (Interactive Web Map)

---

## 🎯 วัตถุประสงค์ของโครงการ
* **การจัดกลุ่มเชิงพื้นที่และเวลา (Spatiotemporal Clustering):** เพื่อจัดกลุ่มจุดความร้อนที่มีความสัมพันธ์กันทั้งในมิติของระยะทางและความต่อเนื่องของเวลา โดยประยุกต์ใช้ algorithm ST-DBSCAN
* **การจำแนกพฤติกรรมไฟ (Fire Behavior Classification):** เพื่อวิเคราะห์และระบุรูปแบบประเภทการเกิดไฟในพื้นที่การใช้ประโยชน์ที่ดินที่แตกต่างกัน (เช่น พื้นที่นาข้าว, พื้นที่ปลูกอ้อย และพื้นที่ป่าสงวน)
* **การพัฒนาเว็บแผนที่แบบ Serverless:** เพื่อสร้างระบบการแสดงผลจำลองแผนที่อินเทอร์แอคทีฟที่แสดงกลุ่มก้อนของไฟ, อัตราการเกิดซ้ำ และปฏิทินการเกิดไฟ (Fire Calendar) เพื่อสนับสนุนการบริหารจัดการเชิงกลยุทธ์

---

## 🛠️ โครงสร้างระบบและเทคโนโลยีที่ใช้ (Tech Stack)

ถูกแบ่งออกเป็น 4 ส่วนการทำงาน (Pipeline) ที่รันเชื่อมต่อกันอย่างเป็นระบบ:

### 🗄️ ส่วนที่ 1 - การเตรียมข้อมูล (Data Preparation)
* **แหล่งข้อมูล (Data Sources):** ข้อมูลจุดความร้อนจากดาวเทียม VIIRS (NASA FIRMS), ข้อมูลขอบเขตการใช้ประโยชน์ที่ดินจังหวัดพิษณุโลก (กรมพัฒนาที่ดิน) และข้อมูลขอบเขตการปกครองจาก HDX
* **การประมวลผลหลัก:** 
  * การแปลงระบบพิกัดทรงกลมโลก WGS84 (EPSG:4326) ไปเป็นระบบพิกัดพื้นที่ UTM Zone 47N (EPSG:32647) เพื่อความแม่นยำในการคำนวณระยะทางในหน่วยเมตร
  * การสร้างดัชนีเวลาในรูปแบบตัวเลขสม่ำเสมอ (`t_days`) เพื่อใช้เป็นฐานสำหรับการคำนวณในมิติเวลา
  * การเขียนฟังก์ชันเพื่อกู้คืนข้อมูลประเภทการใช้ประโยชน์ที่ดินทางการเกษตรที่สูญหายหรือตกหล่นระหว่างชั้นข้อมูล (`RECOVER_TO_AGRI`)

### 🤖 ส่วนที่ 2 - การจัดกลุ่มเชิงพื้นที่และเวลา (ST-DBSCAN Clustering)
* **อัลกอริทึม:** การขยายขีดความสามารถของ Density-Based Spatial Clustering (DBSCAN) เพื่อรองรับพิกัดมิติพื้นที่และเวลาร่วมกัน โดยมีการกำหนดค่า `Eps1` (ระยะทางหน่วยเมตร), `Eps2` (ระยะเวลาหน่วยวัน) และ `MinPts` (จำนวนจุดขั้นต่ำ)
* **การปรับค่าพารามิเตอร์ให้เหมาะสม (Optimization):** ใช้เทคนิค **Grid Search** ร่วมกับการประเมินผลด้วยคะแนน **DBCV Score (Density-Based Clustering Validation)** และมีการใช้ Kneedle Algorithm ร่วมวิเคราะห์เพื่อหาค่าอ้างอิงทางภูมิศาสตร์
* **ไลบรารีที่ใช้:** `scikit-learn`, `geopandas`, `kneed`, `DBCV`, `shapely`, `pyproj`

### 📊 ส่วนที่ 3 - การวิเคราะห์พฤติกรรมไฟ (Behavioral Analysis)
* **การสร้างฟีเจอร์ใหม่:** การสร้างเส้นรอบวงกลุ่มก้อนอัคคีภัยแบบ Convex Hull Polygon ร่วมกับการขยายระยะบัฟเฟอร์ (Buffer) ขนาด 187.5 เมตร เพื่อให้สอดคล้องกับความละเอียดของพิกเซลจากดาวเทียม VIIRS
* **การวิเคราะห์ข้อมูลเชิงลึก:** มีการวิเคราะห์ข้อมูลส่วนที่เป็น Noise (จุดรบกวนที่จัดกลุ่มไม่ได้) ซึ่งพิสูจน์ให้เห็นว่าไฟจากการเผาไร่อ้อยบางประเภททำหน้าที่เป็น Noise ทางสถิติ เนื่องจากมีพฤติกรรมการตัดอ้อยและเผาแยก ไม่ได้ลุกลามเป็นกลุ่มก้อนใหญ่เหมือนไฟป่า
* **แผนที่การเกิดซ้ำ:** พัฒนาระบบตารางกริด (Grid Analysis) ขนาด 1x1 กิโลเมตร เพื่อคำนวณและแปลงค่าความถี่สะสมของการเผาไหม้รายปีตั้งแต่ปี 2020–2025

### 🌐 ส่วนที่ 4 - การสร้างเว็บแผนที่และแดชบอร์ด (Web Map Visualization)
* **โครงสร้างระบบ:** พัฒนาระบบแดชบอร์ดแบบ Static (ไม่ต้องมีระบบหลังบ้าน) โดยใช้วิธีการเขียนโค้ดและฝังโค้ดจำพวก HTML f-strings ลงใน Python Notebook โดยตรงเพื่อประสิทธิภาพสูงสุด
* **เทคโนโลยีฝั่งหน้าบ้าน:** Leaflet.js, การจัดการข้อมูลรูปแบบ GeoJSON, แผนที่ฐานจาก Carto (Light/Satellite)
